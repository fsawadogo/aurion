# Aurion — Infrastructure (Terraform)

Provisions all AWS infrastructure for the Aurion clinical-AI MVP:
VPC, ECS (Fargate + GPU), RDS PostgreSQL, DynamoDB, S3, KMS, Cognito,
AppConfig, CloudWatch.

Region: `ca-central-1` (Canada — required for Quebec Law 25 data
residency). Encoded in `var.region`; do not change without legal review.

## Layout

```
infrastructure/
├── bootstrap/             # ⇐ run this ONCE per AWS account
│   └── (state bucket + DynamoDB lock table + KMS key)
├── backends/              # gitignored — generated by `bootstrap`
│   └── {dev,prod}.s3.tfbackend
├── environments/
│   ├── dev.tfvars
│   └── prod.tfvars
└── (the rest — ecs.tf, rds.tf, s3.tf, … main module)
```

## First-time setup

1. **Bootstrap the remote state** — see `bootstrap/README.md`. Creates
   the S3 + DDB infrastructure that the main module's `backend "s3"`
   block reads from. One-time, ~30 s of `terraform apply`.

2. **Generate the backend config files** — run
   `terraform output -raw backend_dev > ../backends/dev.s3.tfbackend`
   from the bootstrap directory (the README covers it).

3. **Initialize the main module against the dev backend**:
   ```bash
   terraform init -reconfigure -backend-config=backends/dev.s3.tfbackend
   ```

4. **Apply dev**:
   ```bash
   terraform plan  -var-file=environments/dev.tfvars
   terraform apply -var-file=environments/dev.tfvars
   ```

## Switching between environments

```bash
# Dev
terraform init -reconfigure -backend-config=backends/dev.s3.tfbackend
terraform apply -var-file=environments/dev.tfvars

# Prod (human-gated per AURION-CODING-WORKFLOW.md §11)
terraform init -reconfigure -backend-config=backends/prod.s3.tfbackend
terraform apply -var-file=environments/prod.tfvars
```

`-reconfigure` is important — without it, Terraform reuses the cached
backend config from the previous `init`, which is the wrong env's
bucket key.

## Authority

Per `AURION-CODING-WORKFLOW.md` §11 deny-list:

- `terraform apply -var-file=environments/dev.tfvars` is allowed (the
  autonomous loop can run it).
- `terraform apply -var-file=environments/prod.tfvars` is **denied**
  to the loop. Prod applies are human-gated, signed off by the CTO.

## After the first Phase 2 dev apply — domain delegation

`terraform apply` will create the Route 53 hosted zone for
`aurionclinical.com` (only in the env where `manage_root_zone = true`,
which is dev). Until the apex zone's nameservers are delegated from
the registrar, the ACM cert validation in the same apply will hang
for ~15 min and then fail.

To unblock:

```bash
terraform output route53_nameservers
```

Will print 4 ns-* values. Set those at your domain registrar
(GoDaddy, Namecheap, Cloudflare, etc.) as the NS records for
`aurionclinical.com`. Propagation is usually under 10 minutes;
Terraform's `aws_acm_certificate_validation` polls for up to 30 min.

If you registered the domain through Route 53 directly, this step
is skipped — Route 53 auto-delegates to its own zone.
